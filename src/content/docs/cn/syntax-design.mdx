# CroqTile 语法设计

## 简介

CroqTile 是一种嵌入式领域专用语言（eDSL），用于简化加速器硬件上的数据搬运与编排，旨在减轻编写高性能内核工程师的日常负担，包括受硬件能力约束的分块（tiling）等数据搬运复杂性。借助 CroqTile，编程过程更易上手，工程师能更高效地优化这些关键环节。

## 嵌入 C++

CroqTile 是以 C++ 为宿主的 DSL 代码。CroqTile 编译器将 CroqTile 函数做源码到源码的翻译，生成 C++ 代码；会隐式包含 `"croqtile.h"`，使翻译结果能与其他 C++ 代码正确衔接。下面代码片段为一例。

```
// some C++ code

__co__ void croqtile_function() {
  // croqtile code
}

// another C++ code
void foo() {
  croqtile_function();
}

```

注意，CroqTile 函数以 `__co__` 关键字为前缀，函数体内的代码均由 CroqTile 编译器翻译；其余 C++ 代码按声明名直接调用 CroqTile 函数。

此外，为保证 CroqTile 函数实参与其调用方一致，引入了若干 CroqTile 专有类型。因尚有重要编程要素未介绍，相关细节在数据类型说明之后再展开。

## 变量与类型

CroqTile 引入四类基本类型：标量类型（scalar-type）、跨度类型（spanned-type）、整数元组类型（integer-tuple-type / ituple-type）与有界类型（bounded-type），用途各不相同。

- **标量类型**：满足程序控制需求，包含*整数类型*与*布尔类型*。
- **跨度类型**：表示计算所用的数据类型（通常为张量）。除引用原始数据外，*跨度类型*还将数据与由多维范围表示的形状关联；该「多维范围」称为 `mdspan`，对分块等很有用，后文会介绍。
- **整数元组（I-Tuple）类型**：表示一组整数值，常见用法是为多维数据引入边界。
- **有界（整数/ITuple）类型**：用于简化数据（子区域）引用，并与循环结构配合做迭代处理。

四类中，*标量*与 *ituple* 较易理解，可对应通用语言中的元素；*跨度类型*与*有界类型*则更具领域特性。下文详述。

### 标量类型

如前所述，CroqTile 的标量类型包含*整数类型*与*布尔类型*。*整数类型*类似 C++ 的 `int` 或 `int32_t`，为 32 位有符号，取值范围 $-2^{31}$ 到 $2^{31}-1$。下面代码展示其在数据与函数声明中的用法。

```
int a;
__co__ int foo(int b);
```

整数算术、移位与逻辑运算均支持，语法与 C++ 内建运算一致。

CroqTile 不提供无符号标量整数，也不提供 8/16/64 位标量整数的对等物；设计动机是：程序控制未必需要这些类型，通常 32 位有符号整数已足够。

CroqTile 的*布尔类型*类似 C++ 的 `bool`；布尔上的运算及与整数之间的转换，与 C++ 一致。

### 跨度类型

跨度类型是**复合类型**，由**基础类型（fundamental type）**与**多维跨度（multi-dimensional-span，mdspan）类型**组成。*基础类型*与 *mdspan* 单独都不是*完整类型*，即不能单独作为带存储的数据类型。在 CroqTile 中，二者称为*部分类型（partial types）*。

将类型拆成两部分是为了能单独操作 *mdspan*，这在 ML 等场景中至关重要：数据多为多维，程序员常把数据切成更小块并在存储层次间搬运，因此操纵多维数据的*形状*（由 *mdspan* 表示）十分关键。

下文将说明如何定义这些部分类型，以及如何将部分类型组合为完整类型，用于声明/定义多维数据。

#### 部分类型：**mdspan**

与多数类型系统不同，*mdspan* 作为类型实体出现时仍是*部分*的：CroqTile 程序不能把数据定义为单独的 *mdspan* 类型，但可以单独定义 *mdspan* 本身。这样设计是因为循环分块/阻塞往往只关心多维数据的形状，与基础类型无关；因此允许在不关心关联基础类型的情况下操作 *mdspan*，简化与形状相关的编程。

同时，CroqTile 编译器可对 *mdspan* 做类型检查，以便尽早发现错误。

在 CroqTile 中，*mdspan* 由 `[` 与 `]` 括起，维度的上界用 `,` 分隔，例如：

```
mdspan sp : [7, 8];           // defined a mdspan of 2-dimensions
mdspan<4> mds : [a, 3, 4, 1]; // 'a' is an existing integer
d : [c, 4, 28];               // 'd' is not explicit annotated. Type is deduced.
```

如上所示，可用关键字 `mdspan` 显式定义，并可选用尖括号内的总维数；也可不显式标注类型定义 *mdspan*，如上面的 `d`。（注意：`mds` 与 `d` 属于**依赖类型**，因其类型依赖于 `a`、`b` 等表达式的求值；程序员可在运行期提供 `a`、`b`，但会付出一定运行期检查成本。）

对 C++ 程序员而言，可把 *mdspan* 想成多维数组的「特征」：它描述多层范围，例如上例中 `sp` 定义两层范围，分别为 0..6 与 0..7。

定义 *mdspan* 后，可用 `()` 运算符按维度取下标得到整型维度值：

```
sp : [7, 8];
int b = sp(0) + sp(1);   // 'b' equals to 15 (7 + 8)
```

上例中 `sp(0)` 表示 *mdspan* `sp` 第一维的大小；各维大小本质为整数，可用于整数运算。

由此可从已有 *mdspan* 派生新的 *mdspan*：

```
sp : [6, 8];
spn : [1, sp(0)/2, sp(1)/4]; // define a new mdspan from the existing one.
```

该机制便于在 CroqTile 中对多维跨度做分块。由于高性能内核构造中分块几乎总是需要，CroqTile 还提供了更简洁的语法糖：

```
sp : [6, 8];
spn : sp [1, (0)/2, (1)/4];  // spn is defined as [1, 3, 2]
```

语义与上一段等价，写法更简单，符合 CroqTile「在可行前提下用最少代码完成功能」的设计哲学。

**注意：*mdspan* 只能被定义，不能修改已有 *mdspan*；且每个 *mdspan* 只能定义一次。**

除上述按*逐维*方式定义外，CroqTile 还支持其他定义方式；在介绍 *i-tuple* 时会再说明。

#### 完整类型标注

单独使用 *mdspan* 不能定义用于计算的数据。在 CroqTile 函数中，数据定义必须是**完整类型**，由基础类型与 *mdspan* 共同组成。例如：

```
ndims : [20, 15];
f32 [10, 10] d0;
f16 [ndims] d1;
```

CroqTile 支持的基础类型包括：

- 无符号整数：*u8/u16/u32*
- 有符号整数：*s8/s16/s32*
- 浮点：*f16/bf16/f32*

注意在 CroqTile 中 `s32` 与 `int` 不同：`s32` 是基础类型，不能用于完整类型（fully typing）标注。

#### 存储限定符

CroqTile 中带跨度类型的数据通常较大。对这类数据，程序员可在加速器不同内存层级间搬运，以更好利用硬件。

CroqTile 定义三种存储限定符，用于标注所定义数据的存放位置：

- **global**，
- **shared**，
- **local**。

用法示例：

```
ndims : [20, 15];
local f32 [10, 10] d0;
shared f16 [ndims] d1;
```

默认情况下，若无存储限定符，数据视为来自*全局（global）*内存。

### I-Tuple 类型

整数元组是一组整数；通常用作（下标）索引。

定义 *i-tuple* 时，将元素写在 `{` 与 `}` 之间，例如：

```
ituple index = {5, 4, 3, 2, 1};  // It defines a tuple of 5 elements
index = {a, b};  // 'a' and 'b' are existing integers
```

### 对 *mdspan* 与 *i-tuple* 的运算

CroqTile 允许对 *i-tuple* 与 *mdspan* 做特殊运算。下例是对 *mdspan* 做固定分块：

```
sp : [6, 8];
tiling_factor = {3, 2};
spn : sp / tiling_factor;   // spn is defined as [2, 4];

```

CroqTile 中可用此类**元组-跨度运算（Tuple-Span Operations）**定义 *mdspan*，支持的运算包括：

- *mdspan* $/$ *i-tuple*
- *mdspan* $+$ *i-tuple*
- *mdspan* $\%$ *i-tuple*
- *mdspan* $*$ *i-tuple*
- *mdspan* $-$ *i-tuple*

这些运算本质上也可通过逐维定义 *mdspan* 实现，但*元组-跨度运算*有助于写出更可读的代码，这也是 CroqTile 的目标之一。

### 有界类型

有界类型包括**有界标量（Bounded Scalar）**与**有界 ITuple（Bounded ITuple）**。有界标量取值范围为 $[0, \mathrm{ub})$，其中 `ub` 为上界。若整数 `p` 为有界，则须关联具体上界；该关联须在 `parallel-by` 与 `with-in` 等*控制结构*中显式写出，后文介绍。

类似地，*ITuple* 作为一组 *Integer*，也可关联一组上界。在 CroqTile 中，*Bounded ITuple* 与 *mdspan* 值关联，从而确定一组上界。后续章节将给出详细语法。

## 控制结构

CroqTile 在并行化、循环等方面与 C++ 有显著差异。

### 并行区域：`parallel-by` 块

在 CPU 等系统上，可通过异步线程实现并行。CroqTile 则采用单指令多数据（SPMD）模型实现并行，类似部分 OpenMP 并行指令以及 OpenCL/CUDA 等语言。

但构造并行区域的语法不同：采用 C 风格花括号，将并行执行的代码放在 `parallel-by` 块中。

```
parallel p by 6 {
  // SPMD code
}
```

上例展示如何用关键字 `parallel` 与 `by` 创建并行区域：假定有 6 个执行线程，每个线程执行相同的 SPMD 代码，但 `p` 取值不同。若熟悉 CUDA，可把 `p` 类比为线程索引；若更熟悉顺序 C/C++，也可把 `p` 看作 6 次迭代的循环变量（但各次迭代的实际执行顺序不限）。

除并行语义外，`parallel-by` 还有一层含义：语句中的 `p` 是与边界 $[0, 6)$ 关联的整数，我们称之为**有界整数**，而非普通整数。在 `chunkat` 等特殊运算中，需要*有界整数*才能正确工作，因为边界对其计算至关重要。

### `with-in` 块与 `where` 子句

与 `parallel-by` 类似，`with-in` 也可将 *i-tuple* 绑定到 *mdspan*。例如：

```
with index in [10, 10] {
  // index is ituple with 2 elements
}
```

此处 `index` 为含 2 个元素的 *i-tuple*。有时希望两个元素分别命名，可用：

```
with {x, y} in [10, 10] {...}
```

也可同时命名整个 *i-tuple* 及其元素：

```
with index = {x, y} in [10, 10] {...}
```

这种情形下，我们称 `index` 为**有界 ituple（bounded ituple）**。

`with-in` 表面类似 `parallel-by`，但行为不同：重要的一点是 `with-in` **不**隐含并行，块内代码**顺序**执行；其作用主要是创建*有界 ituple*。

此外，可附加 `where` 子句，例如：

```
with {m, n} in [M, N], {n_p, k} in [N_P, K] where n_p <-> n {
  // matmul implements with m,n,K. n_p is no long useful.
}
```

该片段要求在所有迭代中 `n` 与 `n_p` 取值一致，因此在 `with-in` 块内可在需要时用 `n` 替换 `n_p`，反之亦然。该机制对编写多种 AI 内核很有用；程序员应使用 `<->` 建立此类关系。

### `foreach` 块

当 `with-in` 定义了 *bounded-ituple* 后，可对有界 ituple/有界整数做循环。在 CroqTile 中写法很直接：

```
with x in [10] {
  foreach x {
    // do something with each x
  }
}
```

### 异步运算：DMA 语句

除并行执行外，CroqTile 允许一种固定形式的异步运算：DMA 语句。

概念上，DMA 语句与 SPMD 代码**异步**执行，行为类似 CPU 异步线程，但受 DMA 配置约束（CPU 上对异步线程的编程更自由）。

下面是一段基本的 DMA 示例：

```
global f32 [10] g_data;
local f32 [10] l_data;
f = dma.copy.async g_data => l_data;
// ... async operations
wait f;         // explicit wait
```

这里利用数据传输引擎（DTE）发起线性拷贝，将位于全局内存的数据搬到局部内存。注意对 `f` 的赋值：它是异步 DMA 实体的句柄，在 CroqTile 中称为 DMA 操作的 `future`。程序员可写显式同步语句 `wait`，使当前线程在 `future` 完成前阻塞；否则线程与 DMA 并行继续执行。

有时显式定义临时数据较繁琐，CroqTile 提供更简语法：

```
global f32 [10] data;
f = dma.copy data => local;
... f.data;  // retrieve the 'local' data from the future
```

此处不必指定 DMA 的确切目标地址，只需指明目标内存类型；这在许多场景下很有利，程序员常希望少操心便笺式存储（SPM）管理。此例中局部内存分配交给编译器；要取得搬运后的数据，只需调用 `future` 的 `data` 成员。此外，该 dma 未标 `.async`，因此不需要也不能对 `f` 执行 `wait`。

DMA 涉及大量细节，需谨慎编程；应参考 DMA 手册做决策。尽管如此，CroqTile 编译器仍提供大量静态与运行期检查，帮助避免常见错误。

### 有界 ituple/整数与 `chunkat` 运算

`chunkat` 作用于带跨度的数据，在已有数据上产生新的 *mdspan*；某些系统中类似「子视图（subview）」。因 `chunkat` 接受有界 ituple 与有界整数作为参数，在 CroqTile 中命名不同。

```
global f32 [6, 10, 100] data;
parallel p by 6 {
  with index = {x, y} in [10, 10] {
    // for every data move, the stride into 'data' is
    //    stride = p*1000 + x * 100 + y * 10
    //
    // for each chunk, the dimensioned size for the movement is {1, 1, 10}
    f = dma.copy data.chunkat(p, index) => local;
  }
}
```

上例展示 `chunkat` 的典型用法：跨度类型为 `f32 [6, 10, 100]`，`chunkat` 接收整数 `p` 与 ituple `index` 两个参数，假定将数据划分为 $6 \times 10 \times 10$ 块，对应 `p` 与 `index` 相关范围；每次取一块搬到局部存储，沿最低维连续 10 个元素。

## 函数调用与调用 CroqTile 函数

CroqTile 函数**不能**调用另一个 CroqTile 函数；但在 CroqTile 函数内调用 C++ 内核函数是常见用法。

```
void bar() {...}    // C++ kernel function
__co__ void foo() {
  parallel p by 6 {
    call bar();     // Call the C++ function
  }
}

```

上例用 CroqTile 关键字 `call` 调用已有 C++ 函数 `bar`，语义直观。

## CroqTile 与 C++ 之间的参数传递

向 CroqTile 传递参数或反向传递时，需包含头文件 `croqtile.h`。通常将原始 C++ 指针与维度信息组合，构造 CroqTile 的跨度数据。下例说明用法：

```
#include "croqtile.h"

void bar(const float* data, unsigned size) {}

__co__ void foo(f32 mdspan<2> d) {
  parallel p by 6 {
    call bar(d, |d|);     // Call the C++ function
  }
}

void foobar(float* a) {
  foo(croqtile::make_spanview<2>(a, {1, 2}));
}
```

示例使用 CroqTile 工具函数（模板）`make_spanview` 包装数据，不产生拷贝。CroqTile 函数 `foo` 的参数 `f32 mdspan<2>` 与之对应；数据可隐式转换，故可将 `d` 直接作为 C++ 函数 `bar` 的第一个参数；运算符 `|d|` 得到跨度数据 `d` 的总元素个数。二者用于调用 C++ 的 `bar`。

同样，*标量类型*数据也可在 CroqTile 与 C++ 之间传递；但 *ituple* 仅在 CroqTile 函数内部使用。

## 小结

CroqTile 为 SPMD 编程提供了一种新思路：偏好 C++ 风格写法，并嵌入在 C++ 中；其核心是减轻底层细节负担，尤其是通过 DMA 在各内存层之间搬运数据。有时也被称为数据流式 DSL。我们开发该工具以支撑日常高性能内核构建工作，希望程序员少纠结语言构造细节，多关注更高层的概念。
