## 概述

本节将介绍如何用 `with-in` 语句构造*有界变量*，以及如何用 `foreach` 语句创建循环。

## CroqTile 中的循环

在 C++ 中，循环通常用 `for`、`while` 与 `do-while` 构造。这些循环包含终止条件，并常涉及与循环控制相关的变量操作。程序员可以自由定义退出条件、步进方式等，因而循环结构非常灵活。

相比之下，作为面向数据移动的领域特定语言，CroqTile 支持更受约束的循环结构。当前 CroqTile 中的循环构造如下：

```choreo
foreach index {
  // loop body
}
```

此处 `index` 为*有界变量*（见上一节）。若 `index` 的上界为 `6`，则上述代码等价于：

```cpp
for (int index = 0; index < 6; ++index) {
  // loop body
}
```

*有界变量*看似受限，但对数据移动任务已足够，因为通常不会用负下标或越界下标来搬运数据。

在 CroqTile 中，`parallel-by` 语句内定义的*有界变量*是不可变的，不能用于 `foreach` 语句。程序员应改用 `with-in` 语句定义适用于循环的*有界变量*。

## **With-In** 块

### 定义*有界变量*

`with-in` 语句可定义*有界整数*或*有界 ituple*。语法上类似 `parallel-by`，但只定义*有界变量*，不会启动多实例执行。示例如下：

```choreo
with x in 128 {
  // 'x' is a bounded integer
}
with y in [512] {
  // 'y' is a bounded ituple
}
with index in [10, 10] {
  // 'index' is a bounded ituple
}
```

此处 `x` 为有上界 `128` 的有界整数；`y` 与 `index` 均为有界 ituple，上界分别为 `512` 与 `10, 10`。与 `parallel-by` 类似，可为 ituple 元素命名以提高可读性，或同时声明有界 ituple 与对应的有界整数，如下：

```choreo
with {x, y} in [10, 10] {
  // x and y are explicitly named elements of the ituple
}
with index = {x, y} in [10, 10] {
  // 'x', 'y', and 'index' can be used within the block
}
```

与 `parallel-by` 一样，多个 `with-in` 声明可用逗号语法合并，例如：

```choreo
with index = {x, y} in [10, 10], idx in [100, 10] { }
```

这样定义了两个有界 ituple。注意 `index` 与 `idx` 仅能在随后的 `with-in` 块内引用。

## `foreach` 块

### 基本语法

`with-in` 语句定义*有界变量*，随后可用 `foreach` 语句对其迭代。基本语法如下：

```choreo
with index in [6] {
  foreach index {
    // Perform operations with each index
  }
}
```

本例中 `foreach` 块迭代 6 次，**迭代变量** `index` 的值从 `0` 递增到 `5`。

也可以对*有界 ituple* 迭代，例如：

```choreo
with index = {x, y} in [6, 17] {
  foreach index { }
}
```

该代码定义了*有界 ituple* `index`。对 `index` 迭代等价于如下嵌套循环：

```cpp
for (int x = 0; x < 6; ++x)
  for (int y = 0; y < 17; ++y) { }
```

注意循环嵌套的顺序：*有界 ituple* 中*有界变量*的**从左到右**顺序对应循环的**从外到内**嵌套。构造某些代码时，该顺序对正确行为至关重要。

此外，该规则同样适用于 `foreach` 之后的*逗号分隔有界变量列表*：

```choreo
with index = {x, y} in [6, 17], iv in [128] {
  foreach iv, index { }
}
```
此处定义了两个有界变量。随后的 `foreach` 块等价于多级嵌套循环：

```cpp
for (int iv = 0; iv < 128; ++iv)
  for (int x = 0; x < 6; ++x)
    for (int y = 0; y < 17; ++y) { }
```

### 语法糖


```choreo
foreach x in 128 { }
foreach idx in [10, 20] { }
foreach {y, z} in [8, 16] { }
```

上述代码等价于：

```choreo
with x in 128 {
  foreach x { }
}

with idx in [10, 20] {
  foreach idx { }
}

with {y, z} in [8, 16] {
  foreach y, z { }
}
```

### 从有界整数派生循环

某些场景（例如流水线化数据移动）需要调整循环迭代。在 CroqTile 中，可在 `foreach` 中从*有界整数*派生循环。例如：

```choreo
with {x, y} in [6, 17] {
  foreach x, y(1::) { }
}
```

此处使用**范围表达式（Range Expression）** `y(1::)` 派生循环，等价于如下 C/C++ 代码：

```cpp
for (int x = 0; x < 6; ++x)
  for (int y = 1; y < 17; ++y) { }
```

可见 `y` 循环从 `1` 开始。*范围表达式*由*有界变量*及花括号内三个冒号分隔的整数构成，形式如下：

```
  bounded-variable(lower-offset:upper-offset:stride)
```

由此从 `bounded-variable` 派生循环，该 `bounded-variable` 作为循环的*迭代变量*。具体而言：

- *迭代变量*的初值为 `lower-offset` 加上 `bounded-variable` 的下界。CroqTile 中有界整数下界均为 `0`，故 `lower-offset` 决定循环初值。
- 当*迭代变量*大于或等于 `bounded-variable` 的上界加上 `upper-offset` 时循环结束。`upper-offset` 通常为负值。
- 每次迭代结束时，*迭代变量*按 `stride` 递增。

因此，在上例中 `y(1:-1:2)` 会得到类似 `for (y = 0 + 1; y < 17 - 1; y += 2)` 的循环。若*范围表达式*中某字段省略，则使用默认值：`lower-offset` 与 `upper-offset` 默认为 `0`，`stride` 默认为 `1`。

注意*范围表达式*仅适用于*有界变量*。对*有界 ituple* 使用*范围表达式*会在编译时报错。

## 有界变量的取值

理解有界变量关联的取值至关重要。有界变量有两个关键值：

- **当前值（Current Value）**：
    - 在 `foreach` 语句内，当前值由循环迭代决定，因为有界变量作为迭代变量。
    - 在 `foreach` 之外，有界变量的当前值恒为 0。

- **上界值（Upper-Bound Value）**：在 `with-in` 或 `parallel-by` 语句中指定。

程序员在 `foreach` 之外使用有界变量的当前值时可能遇到问题，尤其是在循环结束之后。按定义，有界变量的当前值在 `foreach` 循环外不可变。下面代码说明了这一点：

```choreo
with x in 6 {
  // x's current value is 0
  foreach x {
    // x's current value is either 0, 1, 2, ..., 5
  }
  // x's current value is 0, NOT 6
}
```

本节说明了如何在 CroqTile 中使用 `with-in` 与 `foreach` 定义并迭代*有界变量*——这对数据移动任务至关重要。我们介绍了这些构造的语法与行为，包括如何应用范围操作以修改循环迭代，以及理解有界变量当前值与上界值的重要性。
