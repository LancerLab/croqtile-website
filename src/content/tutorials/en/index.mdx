## What is Croktile?

Croktile is a C++ Embedded Domain Specific Language (EDSL) designed for TileFlow programming on GPUs. It simplifies the process of writing high-performance GPU kernels by providing high-level abstractions for data movement, tiling, and parallelism.

## Why Croktile?

Traditional GPU programming with CUDA requires managing:

- Buffer allocation and deallocation
- Index calculations
- DMA descriptor configuration
- Memory synchronization barriers
- Thread coordination

Croktile eliminates this boilerplate by letting you focus on **what** data moves **where**, rather than **how** to configure the hardware.

## Core Concepts

### 1. Shaped Types

All data in Croktile has a shape:

```cpp
f32 [128, 256] matrix;    // 2D tensor
f32 [M, K] dynamic_input; // Symbolic dimensions
```

### 2. Declarative Data Movement

```cpp
dma.copy input.chunkat(tile_m, tile_n) => shared_memory;
```

### 3. Automatic Safety Checks

The compiler verifies tiling correctness, shape compatibility, and memory bounds at compile time.
